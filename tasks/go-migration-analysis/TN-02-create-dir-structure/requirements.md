# TN-02: Требования к структуре директорий

## 1. Обоснование задачи
Стандартная структура Go-проекта обеспечивает читаемость, масштабируемость и следование лучшим практикам (например, от Google Go Code Review). Это упростит навигацию по коду, разделение на пакеты и интеграцию новых разработчиков.

## 2. Пользовательский сценарий
Разработчик открывает проект и сразу понимает, где находится основной код (`cmd/`), внутренние пакеты (`internal/`), общие утилиты (`pkg/`), что облегчает вклад в проект.

## 3. Требования
- Структура должна следовать рекомендациям Go: `cmd/`, `internal/`, `pkg/`.
- В `cmd/` разместить поддиректорию для основного приложения (например, `server/`).
- В `internal/` — подпакеты для бизнес-логики, инфраструктуры и т.д.
- Добавить README в корне с описанием структуры.

## 4. Критерии приёмки
- [ ] Созданы директории `cmd/`, `internal/`, `pkg/` в `go-app/`.
- [ ] Нет лишних или нестандартных директорий.
- [ ] Структура задокументирована в `go-app/README.md`.
- [ ] `go build ./...` проходит без ошибок после обновления.
