# TN-01: Инициализация Go-модуля

## 1. Обоснование задачи
Чтобы начать миграцию проекта на Go, первым шагом необходимо создать Go-модуль. Это фундаментальное действие, которое закладывает основу для управления зависимостями, версионирования и сборки всего будущего приложения.

## 2. Пользовательский сценарий
Разработчик клонирует репозиторий и может сразу же начать работать с Go-кодом, используя стандартные команды `go build`, `go test` и т.д. CI/CD пайплайны также могут корректно взаимодействовать с проектом.

## 3. Требования
- Создать Go-модуль с именем `github.com/vitaliisemenov/alert-history`.
- Файл `go.mod` должен находиться в корневой директории Go-проекта (`go-app/`).
- В качестве версии Go должна быть указана `1.21` или выше.
- Должен быть создан пустой `main.go` в директории `cmd/server/` для проверки.

## 4. Критерии приёмки
- [ ] В репозитории существует файл `go-app/go.mod`.
- [ ] Команда `go mod tidy` выполняется без ошибок.
- [ ] Команда `go build ./...` успешно собирает проект (пустой `main.go`).
- [ ] В `go.mod` указано корректное имя модуля и версия Go.
