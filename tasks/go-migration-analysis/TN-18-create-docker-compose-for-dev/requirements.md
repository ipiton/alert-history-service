# TN-18: Docker Compose для разработки

## 1. Обоснование
Упрощение локальной разработки с всеми зависимостями.

## 2. Сценарий
Разработчик запускает `docker-compose up` и получает полное окружение.

## 3. Требования
- PostgreSQL с инициализацией схемы.
- Redis для кэширования.
- Приложение с hot-reload.
- Сеть для связи сервисов.

## 4. Критерии приёмки
- [x] docker-compose.yml создан.
- [x] Все сервисы запускаются.
- [x] Приложение подключается к БД и Redis.
- [x] Hot-reload работает.
