# TN-22: Graceful Shutdown

## 1. Обоснование
Корректное завершение работы без потери данных и разрыва соединений.

## 2. Сценарий
При получении SIGTERM приложение завершает обработку и закрывает ресурсы.

## 3. Требования
- Обработка SIGINT, SIGTERM.
- Завершение HTTP запросов.
- Закрытие соединений БД и Redis.
- Таймаут на shutdown.

## 4. Критерии приёмки
- [ ] Signal handler реализован.
- [ ] HTTP сервер gracefully останавливается.
- [ ] Все ресурсы освобождаются.
- [ ] Логирование процесса shutdown.
