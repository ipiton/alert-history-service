# TN-11: Документирование выбора зависимостей

## 1. Обоснование
Фиксация архитектурных решений для будущих разработчиков.

## 2. Сценарий
Разработчик читает ADR и понимает, почему выбраны конкретные библиотеки.

## 3. Требования
- ADR (Architecture Decision Record) для каждого выбора.
- Сравнительная таблица альтернатив.
- Ссылки на бенчмарки.

## 4. Критерии приёмки
- [ ] Создан docs/adr/ директорий.
- [ ] ADR для веб-фреймворка.
- [ ] ADR для БД драйвера.
- [ ] Сводная таблица в README.
