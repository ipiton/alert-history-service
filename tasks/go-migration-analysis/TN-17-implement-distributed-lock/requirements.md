# TN-17: Distributed Lock через Redis

## 1. Обоснование
Координация между несколькими инстансами приложения.

## 2. Сценарий
Только один инстанс обрабатывает определённый алерт одновременно.

## 3. Требования
- Реализация через Redis SET NX.
- Автоматическое освобождение по TTL.
- Защита от deadlock.

## 4. Критерии приёмки
- [ ] DistributedLock реализован.
- [ ] Тесты на конкурентность.
- [ ] TTL и освобождение работают.
- [ ] Документация с примерами.
