# TN-29: LLM Proxy Client POC

## 1. Обоснование
Интеграция с LLM для классификации алертов - ключевая функция.

## 2. Сценарий
Приложение отправляет алерт в LLM proxy и получает классификацию.

## 3. Требования
- HTTP клиент для LLM API.
- Retry логика.
- Timeout handling.
- Mock для тестов.

## 4. Критерии приёмки
- [ ] LLM клиент реализован.
- [ ] Retry и timeout работают.
- [ ] Mock сервер для тестов.
- [ ] Интеграционный тест.
