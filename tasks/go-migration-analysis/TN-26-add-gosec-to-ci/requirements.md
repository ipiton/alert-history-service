# TN-26: Security Scanning с GoSec

## 1. Обоснование
Автоматическая проверка безопасности кода.

## 2. Сценарий
CI запускает gosec и блокирует PR при критических уязвимостях.

## 3. Требования
- GoSec в CI pipeline.
- Конфигурация правил.
- Отчёты в SARIF формате.
- Fail при high severity.

## 4. Критерии приёмки
- [ ] GoSec интегрирован в CI.
- [ ] Конфигурация настроена.
- [ ] Отчёты генерируются.
- [ ] CI блокирует при уязвимостях.
